> [!important]
> **TL;DR** — Before any algorithm can *find* the best portfolio, mathematics has to guarantee that a best portfolio *exists* at all, and the condition that guarantees it is called compactness.
>
> - A set in ordinary space is **compact** when it is both **closed** (it contains its own boundary) and **bounded** (it fits inside some finite box). This is the Heine-Borel theorem, and it is the precise property a feasible region needs.
> - The **Extreme Value Theorem** says a continuous objective on a compact set always attains a maximum and a minimum. That single theorem is the existence guarantee under every portfolio optimizer ever shipped.
> - Optimization breaks in three honest ways: the feasible set is **unbounded** (unlimited leverage, value runs to infinity), it is **open** (a strict inequality, so the best value is only approached), or the objective is **discontinuous** (a fee that jumps, so the optimum falls into the crack).
> - The fix is almost always the same: add real-world constraints — a budget, a leverage cap, weak rather than strict inequalities — which make the feasible set compact and the optimum reappear.
> - The one fact to remember: **closed + bounded + continuous ⇒ the optimum is reached, not merely approached.**

There is a question that sits underneath every line of portfolio code ever written, and almost nobody asks it out loud. You run an optimizer. It hands you a vector of weights and a number — the best expected return for your risk budget, say 12%. You trust it. But step back one inch and ask the rude question: how do you know there *was* a best answer for the machine to find? Maybe the true best was 12.0001%, or 12.5%, or maybe there was no "best" at all and the machine just stopped because it ran out of patience. A solver that returns a number is not the same as a problem that *has* an answer. The number could be a mirage — a value the problem gets closer and closer to but never actually reaches.

This is not a pedantic worry. Quants ship problems every week that have no solution, and the symptom is never an error message — it is an optimizer that returns garbage, or hangs, or spits out absurd 40-times-leveraged weights and calls them optimal. The reason a well-posed portfolio problem is guaranteed an answer, and a badly-posed one is guaranteed *not* to be, comes down to a single property of the feasible region called **compactness**, and a single theorem that pairs it with continuity. This post builds both from nothing — no topology assumed, no analysis background needed — and then shows you, in dollars, exactly when an optimum exists and exactly how it disappears.

![Before and after comparison of a compact feasible set with an optimum and an unbounded set with no optimum](/imgs/blogs/compactness-existence-optima-math-for-quants-1.png)

The figure above is the whole argument in one picture. On the left, a feasible set that is closed and bounded — compact — feeding a continuous objective, and the search lands on an actual best point worth 12%. On the right, an unbounded feasible set feeding the same kind of objective, and the value just keeps climbing with no best point to land on. The difference between a problem that has an answer and one that does not is *entirely* a property of the shape of the region you are searching over, before you run a single iteration. By the end of this article that left-versus-right distinction will be something you can diagnose at a glance, the same way an experienced trader smells a bad fill.

## Foundations: the building blocks

Optimization sounds like it should be about clever algorithms, but the *existence* of an answer is a question about geometry — the shape of the region you are allowed to search and the smoothness of the score you are trying to maximize. To talk about either with any precision we need four ideas, each defined from scratch. A practitioner can skim this; a beginner should not skip it, because every later claim in the post is just these four ideas pushed a little further.

### What a feasible set is

When you optimize a portfolio you are not free to pick any numbers you like. You have rules: the weights must add to 100% of your capital, no single position can exceed 10%, you cannot short certain names, total leverage is capped at two times. Every choice that obeys *all* the rules is called **feasible**, and the collection of all feasible choices — every allowed portfolio at once — is the **feasible set**. Geometrically it is a region in space: if you have two assets, each portfolio is a point $(w_1, w_2)$ on a plane, and the feasible set is some shape on that plane — a line segment, a triangle, a disk — carved out by your constraints. Optimization is then a simple-sounding task: out of all the points in this region, find the one where your objective is biggest.

The whole drama of this post is that *the shape of that region decides whether a biggest point exists at all*. Not the algorithm. The shape.

### What a function and an objective are

A **function** is a machine: feed it an input, it returns an output. Write it $f(x)$, read "f of x," meaning the output the machine $f$ produces from input $x$. In optimization the function is your **objective** — the thing you are scoring. For a portfolio, a common objective is expected return: feed in the weights $w$, get back the portfolio's expected return $f(w)$. You want the feasible point where $f$ is largest. The output is a single number; the input can be a whole vector of weights.

### What "closed" and "bounded" mean, in plain terms

These two words are the heart of everything, so let us define them with no jargon and a concrete picture.

A set is **bounded** if it fits inside some finite box. The interval of numbers from 0 to 1 is bounded — it sits inside the box "between −5 and +5," easily. The set of all positive numbers is *not* bounded: no matter how big a box you draw, there is a number bigger than the box. Bounded means "does not run off to infinity in any direction." For portfolios, bounded means your weights cannot grow without limit — there is a ceiling somewhere.

A set is **closed** if it contains its own boundary — its edge belongs to it. The interval "all numbers from 0 to 1, including 0 and 1" is closed: the two endpoints, which are the boundary, are members of the set. The interval "all numbers strictly between 0 and 1, *not* including 0 or 1" is *not* closed (it is called open): the endpoints are the boundary, and they have been excluded. The distinction feels microscopic — what is the difference between "up to 1" and "up to but not including 1"? — and yet, as we will see in dollars, it is the difference between an optimum that exists and one that does not.

![Stack showing closed and bounded combine into compact which with continuity yields an attained optimum](/imgs/blogs/compactness-existence-optima-math-for-quants-2.png)

The stack above is the logical ladder we are about to climb, read from the bottom up. *Closed* and *bounded* are the two ground-floor properties of a set. Stack them together and you get *compact* — that is the Heine-Borel theorem, the subject of the next section. Add a *continuous* objective on top, and you reach the penthouse: an optimum that is actually *attained*. Each rung depends on the one below it; kick out any rung and the optimum on top can vanish. The rest of the post is just walking up and down this ladder and watching, in concrete dollar examples, what falls apart when a rung is missing.

### What "continuous" means

The last building block is continuity, and the everyday intuition is exact: a function is **continuous** if its graph has no sudden jumps — you can draw it without lifting your pen. If a tiny change in the input can only cause a tiny change in the output, the function is continuous. Expected portfolio return is continuous in the weights: nudge a weight by 0.001 and the expected return moves by a tiny amount, never leaping. A fee schedule that charges 0% below \$1 million traded and suddenly 2% at exactly \$1 million is *not* continuous: at the threshold, an infinitesimal increase in size causes a finite, abrupt drop in your net return. That jump, we will see, is enough to destroy the existence of an optimum even when the feasible set is perfectly compact.

With these four ideas — feasible set, objective, closed-and-bounded, continuous — we have everything we need. The next sections assemble them into the existence guarantee and then, more instructively, break it on purpose.

## 1. The question existence actually answers

Let us be precise about what "an optimum exists" even means, because the word "optimum" hides two very different claims.

When you maximize a function over a set, the cleanest thing you can always talk about is the **supremum** (abbreviated "sup"): the smallest number that is greater than or equal to every value the function takes on the set. The supremum is the *least upper bound* — the lowest ceiling the values never poke through. The supremum almost always exists (as long as the function does not run to infinity). But here is the subtlety that this entire post turns on: **the supremum is a ceiling the values approach; it is not necessarily a value the function ever actually reaches.**

When the supremum *is* reached — when there is a real feasible point where the function equals the supremum — we say the maximum is **attained**, and *that* attained value is the maximum. Existence of an optimum means existence of an *attained* maximum: a flesh-and-blood feasible point you can hand to a trader and say "buy this." A supremum that is never attained is a number with no portfolio behind it — a price tag on an empty shelf.

The whole game is this: the supremum (the ceiling) nearly always exists; the maximum (a point that touches the ceiling) does not. Compactness plus continuity is the exact condition that forces the ceiling to be touched. When people say a problem "has no solution," they almost never mean the supremum is missing — they mean the supremum is finite and perfectly well-defined, but no feasible point ever reaches it. The answer is a limit you can get arbitrarily close to and never grab.

#### Worked example: the bounded closed weight set, where the max-return portfolio exists

Let us make this concrete with the friendliest possible problem, the one where everything works.

You have two assets. Asset A has an expected annual return of 12%; asset B has an expected annual return of 6%. You will put a fraction $w$ of your \$1,000 into A and the remaining $1-w$ into B. The rules: you must be fully invested (weights sum to 100%, automatic here), and you cannot short or use leverage, so $w$ runs from 0 to 1, endpoints included. The feasible set is the closed interval $[0, 1]$ — closed because it includes both 0 and 1, bounded because it sits between 0 and 1.

The objective is the portfolio's expected return:

$$ f(w) = 12\% \cdot w + 6\% \cdot (1-w) = 6\% + 6\% \cdot w. $$

This is a straight line in $w$, rising as $w$ grows, and it is continuous (no jumps). To maximize a rising line on $[0,1]$ you push $w$ as far right as the rules allow, which is $w = 1$. Because 1 is *included* in the set, you can actually choose it. The maximum is attained at $w = 1$:

$$ f(1) = 6\% + 6\% \cdot 1 = 12\%. $$

In dollars: put the whole \$1,000 into asset A and your expected return is \$120 on the year, the best the feasible set allows. The maximizing portfolio is a real point, $w = 1$, and the maximum value, 12%, is a real number a real portfolio achieves.

Notice every ingredient of the guarantee was present: the set $[0,1]$ is closed (contains its endpoints) and bounded (fits in a box), so it is compact; the objective is continuous; and the conclusion is that the maximum is *attained at an actual feasible point*. This is the Extreme Value Theorem doing its job, which we will state formally in Section 3. The one-sentence intuition: when the region is closed and bounded and the score is smooth, the best portfolio is a real portfolio you can actually buy.

## 2. Compactness: closed plus bounded

Now we name the property that made the friendly example work. In ordinary $n$-dimensional space — the space $\mathbb{R}^n$ where every portfolio is a point and every constraint is a wall — a set is called **compact** exactly when it is both closed and bounded. That equivalence is a celebrated result called the **Heine-Borel theorem**, and it is the workhorse of this entire post.

> The Heine-Borel theorem: In $\mathbb{R}^n$, a set is compact if and only if it is closed and bounded.

We should be honest about one thing, because the honesty is the point of a math-for-quants series. The "real" definition of compactness, the one that works in stranger infinite-dimensional spaces, is not "closed and bounded" — it is a more abstract statement about open covers (every collection of open sets that covers the set has a finite sub-collection that still covers it). For the spaces quants actually optimize over — vectors of portfolio weights, which live in finite-dimensional $\mathbb{R}^n$ — Heine-Borel says you can forget the abstract version entirely and just check two concrete boxes: is the set closed, and is it bounded? If both, it is compact, and the existence machinery switches on. That is a genuine gift: a deep topological property reduced to two things you can eyeball.

### Closed sets: the boundary belongs

Let us pin down "closed" with the precision the worked examples will demand. A set is **closed** if it contains all of its **limit points** — every point you can get arbitrarily close to using points of the set. Concretely: if you can build a sequence of points inside the set that marches toward some target point, then for the set to be closed, that target must itself be in the set. The interval $[0,1]$ is closed because any sequence inside it that converges (say $0.9, 0.99, 0.999, \dots$ marching toward 1) converges to a point (here 1) that is also in the set. The interval $[0,1)$ — everything up to but excluding 1 — is *not* closed, because that same sequence marches toward 1 and 1 has been kicked out.

In portfolio language, closed means your constraints are **weak inequalities**: "weight $\le$ 10%," "leverage $\le$ 2," "weight $\ge$ 0." Weak inequalities include the boundary, so the set is closed. **Strict** inequalities — "weight $<$ 10%," "leverage $<$ 2" — exclude the boundary and make the set open, which, as the third worked example will show, is exactly how a perfectly finite optimum becomes unreachable.

### Bounded sets: it fits in a box

Bounded is the easier of the two. A set is **bounded** if there is some finite number $R$ such that every point of the set sits within distance $R$ of the origin — the whole set fits inside a ball (or box) of finite radius. Portfolio weights that sum to 1 with each weight between 0 and 1 are automatically bounded: no weight exceeds 1, so no point is far from the origin. The danger to boundedness is always the same in finance: **leverage with no cap**. If you may borrow without limit, your weights can grow toward infinity, the set runs off the edge of every box, and boundedness fails. We will watch that failure cost an optimum in the second worked example.

![Matrix showing closed bounded compact and optimum-exists status for four feasible sets](/imgs/blogs/compactness-existence-optima-math-for-quants-3.png)

The matrix above is a diagnostic table you can run in your head for any feasible set. Each row is a different constraint regime; the columns ask, in order, is it closed, is it bounded, is it therefore compact, and does an optimum exist. Read it and the pattern jumps out: the *only* row that gets a clean "yes" all the way across is the one that is both closed and bounded. Weights restricted by a strict inequality lose closedness and with it the attained optimum (you are left with only a supremum). Uncapped leverage loses boundedness and the value runs away. And the empty feasible set — when your constraints contradict each other and no portfolio satisfies all of them — has no feasible point to optimize at all, a failure mode worth naming because it shows up the moment a risk limit and a return target are mutually incompatible. Compactness is the *conjunction* of two properties, and losing either one is enough.

#### Worked example: unlimited leverage, where expected return has no maximum

Here is the cleanest way to break existence by losing boundedness, and it is a mistake that has been made on real desks.

Keep asset A at a 12% expected return. Now suppose your broker offers you leverage and you write the optimization with no cap on it. Let $L$ be your leverage — the multiple of your own capital you deploy in asset A, funded by borrowing the rest. Borrowing costs 4% (the financing rate). Your expected return on your own \$1,000 of equity is the leveraged return on A minus the cost of the borrowed portion:

$$ f(L) = 12\% \cdot L - 4\% \cdot (L - 1) = 8\% \cdot L + 4\%. $$

The first term is the gross return on $L$ times your capital invested in A; the second subtracts financing on the borrowed multiple $L-1$. Simplify and the expected return is $8\% \cdot L + 4\%$, a straight line rising in $L$ with slope 8%. Every extra turn of leverage adds 8 percentage points of expected return, because A earns 12% and the borrowing only costs 4%, leaving an 8% spread you can lever as hard as you like.

Now ask for the maximum. The feasible set is "$L \ge 1$ with no upper bound" — the half-line from 1 to infinity. It is *closed* (it includes its left endpoint and contains its limit points) but it is *not bounded* (it runs to infinity). So it is **not compact**. And the consequence is immediate: there is no best $L$. At $L = 10$ you expect $8\% \cdot 10 + 4\% = 84\%$, or \$840 on \$1,000. At $L = 100$ you expect \$8,040. At $L = 1{,}000$, over \$80,000. The supremum is *infinity* — there is no finite ceiling — so there is certainly no attained maximum. The "optimal" leverage runs away to infinity, and a naive solver handed this problem either returns the largest number its bounds allow (a lie dressed as an answer) or diverges.

The point is not that infinite leverage is risky (it is, lethally, but that is a different post). The point is mathematical: by leaving the feasible set unbounded you destroyed compactness, and with it the very existence of an optimum. The expected-return objective was perfectly continuous; that was never the problem. Boundedness was. The one-sentence intuition: a problem with no ceiling on the choices can have no best choice — the search just climbs forever.

![Before and after comparison of uncapped leverage running away versus a leverage cap restoring a finite optimum](/imgs/blogs/compactness-existence-optima-math-for-quants-7.png)

The before-after above is the fix for exactly this pathology, and it is the fix you should reach for every time. On the left, the uncapped problem: borrow without limit, the return scales forever, there is no best leverage, the optimum is infinite. On the right, the same problem with one extra line — leverage capped at two times. That single weak inequality, $L \le 2$, makes the feasible set bounded, hence compact, and the optimum snaps back into existence. With the cap, the best feasible leverage is $L = 2$, giving $8\% \cdot 2 + 4\% = 20\%$ gross of the worked-example's stated 18% net once you carry a small drag — a real, finite, attainable number. We will use this exact "add a constraint to restore compactness" move again and again; it is the single most important practical lesson in the post, and it is why real mandates *always* carry leverage limits. This connects directly to the discussion of why a 100%-equity allocation is rarely optimal once real constraints bind, covered in [why not 100% equities](/blog/trading/quantitative-finance/jpm-why-not-100-equities).

## 3. The Extreme Value Theorem: the existence guarantee

We now have compactness. The theorem that turns it into a guarantee about optima is one of the most useful results in all of applied mathematics, and it has a name worth saying out loud: the **Extreme Value Theorem**, sometimes called the Weierstrass theorem after Karl Weierstrass, who put the foundations of analysis on firm ground in the nineteenth century.

> The Extreme Value Theorem: If $f$ is a continuous function on a non-empty compact set $K$, then $f$ attains both a maximum and a minimum on $K$ — there exist points $x_{\max}$ and $x_{\min}$ in $K$ where $f$ reaches its largest and smallest values.

Read that carefully, because every word is load-bearing. *Continuous* — no jumps. *Compact* — closed and bounded. *Non-empty* — there is at least one feasible point. Grant all three and the conclusion is not "the supremum exists" (a weak claim) but "the maximum is *attained at an actual point of $K$*" — there is a real $x_{\max}$ you can name and hand over. The theorem converts a property of the *shape* of the region (compactness) and a property of the *score* (continuity) into a property of the *answer* (it exists and is reached). That is exactly the gap between a mirage and a portfolio.

![Tree of the requirements for a guaranteed optimum branching into compact set and continuous objective](/imgs/blogs/compactness-existence-optima-math-for-quants-4.png)

The tree above lays out the theorem's hypotheses as a checklist you can audit any optimization against. The root is the thing you want: a guaranteed optimum. It rests on two branches — a compact feasible set and a continuous objective — and each branch has its own leaves. The compact branch needs the set closed *and* bounded; the continuous branch needs no jumps in the payoff and a non-empty set to begin with. The discipline this tree enforces is simple and powerful: before you trust an optimizer's output, walk every leaf and confirm it. A failure at any single leaf — an open constraint, an uncapped variable, a jump in the fee, an infeasible set — and the root guarantee evaporates, no matter how confident the solver looks. Most "my optimizer gave nonsense" stories are a leaf of this tree quietly failing.

### Why continuity is non-negotiable

It is worth seeing *why* continuity cannot be dropped, because the reason is the same reason a jumping fee breaks things. Suppose your objective is continuous everywhere on a compact set *except* at one point, where it leaps up. The supremum of the function might be that leaped-up value — but if the jump is the kind where the function gets close to a high value without ever equaling it (an open jump), the supremum is never attained, and the maximum does not exist even though the set is compact. Continuity is the property that prevents the function's values from "escaping" the way the set's points cannot. Compactness pins the *domain* so it cannot run off; continuity pins the *range* so it cannot leap away. You need both clamps. The fourth worked example makes this dollar-concrete.

### Why closedness specifically matters

There is a clean way to see the role of closedness through the lens of the theorem. A continuous function maps compact sets to compact sets — feed a closed-and-bounded region through a continuous machine and the set of outputs is itself closed and bounded. A closed-and-bounded set of *numbers* on the real line is an interval like $[a, b]$ that *includes its endpoints*. The maximum of the function is the right endpoint $b$ of that output interval — and because the output set is closed, $b$ is included, so some input actually produces it. Drop closedness on the input side and the output set can be a half-open interval $[a, b)$ whose top value $b$ is excluded: a ceiling the function approaches but never hits. That is the whole mechanism, and it is why "the boundary belongs to the set" is not a technicality but the linchpin.

#### Worked example: the open constraint, where the best value is a sup that is never attained

This is the most instructive failure in the post, because nothing runs to infinity and nothing jumps — and yet the optimum still does not exist. It is the pure consequence of an *open* set.

Same two assets: A at 12% expected return, B at 6%. You will hold a fraction $w$ in A. But now a risk officer imposes a *strict* concentration limit: no single position may be 50% *or more* of the book. In symbols, $w < 0.5$ — a strict inequality, boundary excluded. Combined with no shorting ($w \ge 0$), the feasible set is the half-open interval $[0, 0.5)$: it includes 0 but *excludes* 0.5.

The objective is the same rising line as before:

$$ f(w) = 6\% + 6\% \cdot w. $$

To maximize a rising line you push $w$ as far right as allowed. But the right edge, $w = 0.5$, is *not in the set*. You can choose $w = 0.49$ (return $6\% + 6\% \cdot 0.49 = 8.94\%$), or $w = 0.499$ (return $8.994\%$), or $w = 0.4999$ (return $8.9994\%$). Each step gets you closer to 9% but you can never reach it, because reaching 9% would require $w = 0.5$, which the strict limit forbids. The **supremum is 9%** — that is the least upper bound, the ceiling — but it is **never attained**. There is no best feasible portfolio: for any portfolio you propose with $w = 0.5 - \varepsilon$, I can propose a strictly better one with $w = 0.5 - \varepsilon/2$, forever.

In dollars: on \$1,000 the supremum is \$90 of expected return, but every actual feasible portfolio earns strictly less than \$90, and there is no "least less" — you can earn \$89.94, \$89.99, \$89.999, with no best. The feasible set was bounded (it fits between 0 and 0.5) and the objective was continuous, but the set was *not closed* (it excluded its boundary point 0.5), so it was not compact, and the Extreme Value Theorem does not apply. The optimum genuinely does not exist.

The fix is a one-character change with a real economic meaning: make the constraint *weak*, $w \le 0.5$. Now the boundary 0.5 is included, the set is the closed interval $[0, 0.5]$, it is compact, and the maximum is attained at $w = 0.5$ for an expected return of exactly $6\% + 6\% \cdot 0.5 = 9\%$, or \$90. A real risk officer almost always means "$\le 50\%$" anyway, so the fix is also the honest reading of the constraint. The one-sentence intuition: a strict inequality removes the very point where the optimum wants to live, leaving you forever reaching for a value you can see but never touch.

![Before and after comparison of a closed constraint reaching the maximum versus an open constraint with an unattained supremum](/imgs/blogs/compactness-existence-optima-math-for-quants-5.png)

The before-after above contrasts the closed and open versions of that exact constraint, and it is worth lingering on because the two pictures look almost identical and behave completely differently. On the left, the weak constraint "$w \le 0.5$": the boundary 0.5 is *included*, you pick $w = 0.5$, and you hit the maximum return cleanly. On the right, the strict constraint "$w < 0.5$": the boundary 0.5 is *excluded*, you approach but never reach it, and what looked like a 9% optimum is only a supremum, a price tag with no portfolio behind it. The difference between the two columns is a single symbol — $\le$ versus $<$ — and it is the difference between an optimization that has an answer and one that does not. This is why production portfolio solvers are written with weak inequalities; the relationship between binding constraints and the optimum is exactly what the [Lagrangian and KKT conditions](/blog/trading/math-for-quants/lagrangian-kkt-conditions-math-for-quants) formalize, where the constraint that binds at the boundary carries a shadow price precisely *because* the boundary is reachable.

#### Worked example: a fee that jumps at a threshold, breaking existence through discontinuity

The last way to break existence keeps the feasible set perfectly compact and attacks the *objective* instead, by making it discontinuous. This is the subtlest failure and the most common in real markets, because fee schedules, tax brackets, and rebate tiers are all discontinuous by design.

You are sizing a trade of notional $x$, ranging from \$0 to \$2 million — a closed, bounded, compact interval $[0, 2{,}000{,}000]$. Your expected alpha (edge, the profit your signal predicts) on the trade is 1% of notional, so the gross profit is $0.01 \cdot x$. The feasible set could not be more compact: a single closed interval. The trap is in the fee schedule, and the *direction* of the jump is what decides whether the optimum survives, so we take the punishing-but-realistic direction: the broker charges a flat \$500 for any trade *below* \$1 million, but the fee jumps *up* to a flat \$3,000 at \$1 million and above — a tier where crossing into "large trade" territory triggers extra reporting, clearing, and market-impact costs. The fee gets *worse* exactly when you cross the threshold, which is precisely the configuration that hides the optimum. Net profit is gross alpha minus the tiered fee:

$$ \text{net}(x) = \begin{cases} 0.01\,x - 500 & \text{if } x < 1{,}000{,}000 \\ 0.01\,x - 3{,}000 & \text{if } x \ge 1{,}000{,}000. \end{cases} $$

Look at what happens around the threshold. Just below \$1 million, at $x = 999{,}999$, net profit is $0.01 \cdot 999{,}999 - 500 = \$9{,}499.99$. Exactly at \$1 million, net profit drops to $0.01 \cdot 1{,}000{,}000 - 3{,}000 = \$7{,}000$ — a cliff. As $x$ climbs from 0 toward \$1 million the net profit rises toward \$9,500 but never reaches it (at $x = 1{,}000{,}000$ the rule switches and the value falls off the cliff). The supremum of net profit on the "below threshold" piece is \$9,500, **approached as $x \to 1{,}000{,}000$ from below but never attained**, because the only point where net would equal \$9,500 is $x = 1{,}000{,}000$ itself, where the fee jumps and net is actually \$7,000.

So even though the feasible set $[0, 2{,}000{,}000]$ is compact, the *discontinuity* in the objective at \$1 million means the maximum over the lower piece is a supremum that is never attained. The Extreme Value Theorem's continuity hypothesis is violated, and its conclusion fails. A solver hunting for the max will chase $x$ ever closer to \$1 million, profit creeping toward \$9,500, and never settle — or it will land on \$999,999.99 and call it optimal, which is fragile nonsense (one more dollar of size and you lose \$2,500).

The fix mirrors the others: restore continuity, or restructure the feasible set so the optimum lives at an attained point. In practice you either (a) split the problem into two compact sub-problems — optimize separately on $[0, 1{,}000{,}000)$ closed off at a hair below the threshold and on $[1{,}000{,}000, 2{,}000{,}000]$ — and compare the attained maxima of each, or (b) model the fee as continuous (a smoothed schedule) so the cliff disappears. Splitting reveals the honest answer: on the upper compact piece the max is attained at $x = 2{,}000{,}000$ with net $0.01 \cdot 2{,}000{,}000 - 3{,}000 = \$17{,}000$, which beats anything below the threshold, so the true optimum is the full \$2 million trade worth \$17,000. The discontinuity did not make the problem unanswerable — it made it *not solvable by naively maximizing*, and the cure was to carve the domain into pieces on which continuity (and hence the theorem) holds. The one-sentence intuition: a jump in the score can hide the best value in a crack between the pieces, and the fix is to optimize each smooth piece on its own compact set and take the winner.

## 4. Why constraints make the problem solvable

There is a beautiful inversion here that is worth stating plainly, because it overturns the beginner's instinct. A newcomer thinks constraints are *obstacles* — annoying limits that can only shrink your opportunity and lower your best achievable return. And it is true that adding a constraint can never *raise* the maximum (a smaller feasible set cannot contain a better point than a larger one that includes it). But constraints do something the unconstrained problem often cannot: they can make the optimum *exist at all*.

The unconstrained "maximize expected return" problem is pathological precisely because expected return is unbounded — pile on leverage and it climbs forever, so there is no maximum, as the second worked example showed. The *constrained* problem — "maximize expected return subject to a budget and a leverage cap and position limits" — carves the feasible set down to a compact region, and on a compact region a continuous objective is *guaranteed* a maximum by the Extreme Value Theorem. The constraints are not the enemy of the optimum; in the unbounded case they are its *parents*. Without them there is nothing to find.

![Pipeline from adding budget and bounds to a compact set to continuity to an attained optimum and solver output](/imgs/blogs/compactness-existence-optima-math-for-quants-6.png)

The pipeline above traces exactly how a real portfolio problem becomes well-posed, left to right. You start by adding budget and bound constraints. Those carve the feasible set into a compact region. You feed that compact region into a continuous objective. The Extreme Value Theorem then guarantees a maximum exists. And only because it exists does the solver return a real vector of weights you can trade. Every arrow is a logical "therefore," and the chain is only as strong as its weakest link: skip the constraints and the first box never produces a compact set, so the guarantee at the end never fires. This is the deep reason every production mandate is *constrained* — not merely for risk management, but so that the optimization is mathematically guaranteed an answer in the first place. The mechanics of solving such a constrained quadratic problem, once existence is secured, are the subject of [mean-variance optimization and the efficient frontier](/blog/trading/math-for-quants/mean-variance-efficient-frontier-math-for-quants).

### The budget constraint as a compactness machine

The single most important constraint in all of portfolio optimization is the **budget constraint**: the weights must sum to one, $\sum_i w_i = 1$. By itself this is a *closed* condition (it is an equality, and equalities include their boundary trivially), but it does not bound the weights — you could have $w_1 = 1{,}000{,}000$ and $w_2 = -999{,}999$ and still sum to one. So the budget constraint alone gives closedness but not boundedness. Pair it with a **no-shorting** or **leverage-cap** constraint, though, and the weights are pinned: if every weight is between 0 and 1 and they sum to 1, the feasible set is the *simplex* — a closed, bounded, compact region (a triangle for three assets, a line segment for two). The simplex is the canonical compact feasible set of finance, and it is compact precisely because the budget equality (closed) meets the non-negativity bounds (bounded together with the budget). Mean-variance optimization lives on this simplex, and that is *why* it always has a solution.

### Mean-variance optimization always has an answer — here is why

Consider the workhorse problem, maximize $w^\top \mu - \tfrac{\gamma}{2} w^\top \Sigma w$ subject to $w^\top \mathbf{1} = 1$ and $0 \le w_i \le 1$. The objective — expected return minus a risk penalty — is continuous (it is a smooth quadratic in $w$). The feasible set — the simplex — is compact (closed budget equality plus bounded weights). Continuous objective on a compact set: the Extreme Value Theorem fires, and a maximum is *guaranteed to be attained*. The optimizer is not hoping to find an answer; it is guaranteed one exists before it takes a single step. That guarantee is the quiet foundation under every mean-variance allocation, every risk-parity portfolio, every constrained Markowitz solve a desk runs. Take away the bounds and add unlimited leverage, and the guarantee can vanish — which is exactly why it almost never is taken away in practice.

#### Worked example: adding a budget constraint to rescue a runaway three-asset problem

Let us put the rescue together with numbers. You have three assets with expected returns 12%, 9%, and 5%. The *unconstrained* "maximize expected return with leverage" problem has no solution: pour infinite leverage into the 12% asset and the return runs to infinity, exactly as in the second worked example.

Now impose the two constraints that make finance finance: a budget ($w_1 + w_2 + w_3 = 1$, fully invested) and no shorting ($w_i \ge 0$). The feasible set is now the simplex — every $(w_1, w_2, w_3)$ of non-negative weights summing to 1. It is compact: closed (budget equality and weak $\ge 0$ inequalities all include their boundaries) and bounded (no weight can exceed 1). The objective, expected return $12\% w_1 + 9\% w_2 + 5\% w_3$, is continuous.

By the Extreme Value Theorem a maximum *exists*, and because the objective is a rising linear function the maximum sits at a corner of the simplex — the corner $w = (1, 0, 0)$, all-in on the 12% asset. The attained maximum is exactly 12%, or \$120 on \$1,000. The same return that ran to infinity in the unconstrained problem is pinned to a finite, attained 12% the moment the budget and non-negativity constraints make the set compact. The constraints did not merely cap the answer; for the *leveraged* version they conjured an answer out of a problem that had none. The one-sentence intuition: a budget that forces your weights to sum to one, plus a floor that forbids shorting, fences the search into a compact region where a best portfolio is mathematically guaranteed to exist.

## 5. Sequential compactness: why the search converges

There is one more face of compactness that deserves its own section, because it is the version that actually explains *why a solver can find the optimum* and not merely that one exists. It is called **sequential compactness**, and in $\mathbb{R}^n$ it is equivalent to ordinary compactness (closed and bounded) — another gift of Heine-Borel.

> Sequential compactness: A set $K$ is sequentially compact if every sequence of points in $K$ has a subsequence that converges to a point *also in $K$*.

Unpack that with the intuition first. A **sequence** is an endless list of points — the successive guesses an optimizer makes as it iterates. A **subsequence** is a thinned-out version of that list: you keep some of the guesses and drop the rest, preserving their order. **Convergence** means the points settle down toward a single limit point, getting arbitrarily close and staying close. Sequential compactness says: no matter how an infinite list of points wanders around inside a compact set, you can always extract a thinned-out version that homes in on a single point — and that point is guaranteed to be inside the set, not on some excluded boundary.

### The Bolzano-Weierstrass theorem in plain language

The reason this holds in a box is a classic result, the **Bolzano-Weierstrass theorem**: every bounded sequence in $\mathbb{R}^n$ has a convergent subsequence. The picture is a pigeonhole argument. Take a bounded sequence — infinitely many points crammed into a finite box. Cut the box in half; one half must contain infinitely many of the points (you cannot fit infinitely many points into finitely many halves without piling up somewhere). Keep that half, cut *it* in half, again one piece holds infinitely many points. Repeat forever and the nested boxes shrink to a single point; pick one sequence-point from each box and you have a subsequence converging to that point. Boundedness is what makes the boxes finite to begin with; closedness is what guarantees the limit point you converge to is actually *in the set* rather than an excluded edge.

### Why a "maximizing sequence has a convergent subsequence" gives the optimum

Here is the punchline, and it is the proof of the Extreme Value Theorem in two sentences. Take a **maximizing sequence**: a list of feasible points whose objective values climb toward the supremum (such a sequence always exists by the definition of supremum — you can always find points getting closer to the ceiling). The feasible set is compact, so by sequential compactness this sequence has a subsequence converging to some point $x^\star$ *in the set*. Because the objective is continuous, the objective values along that subsequence converge to $f(x^\star)$ — but they were also converging to the supremum, so $f(x^\star)$ *equals* the supremum. Therefore $x^\star$ is a feasible point that attains the supremum: the maximum exists and is reached at $x^\star$. That is the entire mechanism, and you can see precisely where each hypothesis is spent — boundedness and closedness deliver the convergent-in-the-set subsequence, continuity carries the objective values to the limit. Remove compactness and the subsequence might converge to a point outside the set (or not converge at all); remove continuity and the limit of the values need not be the value at the limit. Both clamps, both spent, optimum secured.

This is also why optimization algorithms *terminate sensibly* on compact feasible sets: the iterates live in a compact region, so they cannot escape to infinity, and a convergent subsequence is always available to home in on a genuine optimum. On a non-compact set the iterates can march off toward an unattained supremum forever, which is the algorithmic shadow of "no optimum exists."

It is worth being precise about what sequential compactness does and does not promise, because it is easy to over-read. It does *not* say the whole sequence of an optimizer's iterates converges — a badly tuned solver can oscillate forever, visiting good points and bad ones in turn. What it guarantees is weaker but exactly enough: *some* thinned-out version of the iterates converges, and the limit is in the set. For an existence proof that is all you need — you are proving a best point *is there*, not that any particular algorithm walks straight to it. The job of finding it efficiently is a separate engineering problem (gradient steps, interior-point methods, active-set solvers), and those methods earn their convergence guarantees by adding structure like convexity on top of the bare existence that compactness provides. Existence first, then efficiency: compactness secures the former, convexity and smart steps secure the latter. The same convergence logic underlies sequential decision problems like repeated betting, where the long-run growth-optimal fraction is an attained interior optimum precisely because the feasible bet sizes form a compact interval — see [the Kelly criterion and sequential betting](/blog/trading/quantitative-finance/kelly-criterion-sequential-betting-quant-interviews).

## Common misconceptions

**"If the optimizer returned a number, the optimum exists."** No. A solver returns a number when it stops iterating, which can happen for reasons unrelated to having found an attained optimum: it hit an iteration cap, it bumped against an artificial bound you set, or it converged to the *supremum* of a problem that never attains it. The number can be a value the true problem only approaches. Existence is a property of the *problem* (compact set, continuous objective), not of the solver's willingness to print something.

**"Bounded is enough — if my weights can't blow up, I'm fine."** Bounded is half of compactness. A bounded-but-open set, like "weight strictly below 50%," is not compact, and its optimum can be an unattained supremum, as the third worked example showed. You need closed *and* bounded. The most common real-world violation is writing a strict inequality (`<`) where you meant a weak one (`<=`); it costs you the boundary point, which is usually exactly where the optimum lives.

**"Compactness is abstract topology with no practical bearing on my code."** In finite-dimensional space — where all portfolio-weight optimization lives — Heine-Borel reduces compactness to two checks any practitioner can run: is the feasible set closed (weak inequalities, equalities, includes its boundary)? Is it bounded (some cap pins every variable)? Those two checks are a thirty-second audit that predicts whether your problem has an answer, before you write a line of solver code. That is as practical as math gets.

**"A discontinuous objective just makes the problem harder, not unsolvable."** A discontinuity can make the *naive* maximization unsolvable — the optimum can fall into the crack of a jump, attained nowhere, as the fee example showed. The problem is not that it is hard; it is that the Extreme Value Theorem's continuity hypothesis fails and the existence guarantee with it. The cure is structural: split the domain into pieces on which the objective *is* continuous, optimize each on its own compact set, and compare. Throwing a bigger solver at a discontinuous objective does not restore existence.

**"Adding constraints can only hurt my expected return, so fewer constraints is always better."** Adding a constraint can never raise the maximum, true — but for an unbounded problem there *is* no maximum to begin with, and the constraint is what brings one into existence. Constraints are simultaneously a ceiling on how good your answer can be and the very thing that guarantees you have an answer. A trader who strips out every constraint to chase a higher number can strip the problem of its solvability entirely.

**"The supremum and the maximum are the same thing."** They coincide *only when the maximum is attained*. The supremum is the least upper bound — the lowest ceiling the values never exceed — and it almost always exists for a problem that does not run to infinity. The maximum is an actual achieved value at an actual feasible point, and it exists only under compactness-plus-continuity. Confusing the two is the single most common conceptual error behind "my optimal portfolio doesn't make sense": you found a supremum and treated it as a portfolio.

## How it shows up in real markets

**Uncapped leverage and the LTCM-style blowup.** Long-Term Capital Management famously ran leverage estimated around 25-to-1 on balance sheet and far higher through derivatives, on positions whose "optimal" size, under their models' assumptions, scaled with leverage. When the spread trades looked riskless, the model wanted *more* size, and with no hard cap the implied optimum kept climbing — the unbounded-feasible-set pathology in the wild. When correlations spiked in August 1998 the positions that were supposed to be nearly risk-free moved together, and the leverage that the unconstrained optimum had loaded turned a 4-sigma move into insolvency. The lesson in this post's language: a feasible set with no leverage bound has no honest optimum, and the "optimal" leverage a model reports for such a set is a number with no finite ceiling behind it. The fix the entire industry adopted — hard leverage and concentration limits — is precisely the act of making the feasible set compact.

**Risk limits written as strict versus weak inequalities.** Production risk systems encode limits like "single-name exposure ≤ 5% of NAV." Whether that is coded as `<=` or `<` is not cosmetic. A strict `<` makes the constraint set open at the binding face; an optimizer that wants to sit *exactly* at 5% (often the return-maximizing choice when the constraint binds) then chases an unattained supremum and either fails to converge or returns 4.9999%, leaving free expected return on the table and producing jittery, unstable weights run-to-run. Desks that have debugged "why does my optimizer give slightly different weights every time" have often found a strict inequality where a weak one belonged — the open-set pathology of the third worked example, costing real basis points.

**Transaction-cost and tax cliffs.** Brokerage fee schedules, exchange rebate tiers, and capital-gains tax brackets are all discontinuous: a threshold where the cost or tax rate jumps. An optimizer that maximizes after-cost or after-tax return over a continuous size variable faces exactly the discontinuous-objective pathology — the optimum can sit infinitesimally below a threshold, attained nowhere, fragile to a single dollar of size. Real execution and tax-aware optimizers handle this by piecewise optimization: solve on each tier (each a compact sub-interval), then compare attained optima across tiers. The 2017-2018 wave of tax-loss-harvesting products had to grapple with this directly, because the wash-sale rule and bracket thresholds inject discontinuities into what looks like a smooth after-tax objective.

**Why mandates are always constrained.** Every institutional investment mandate — pension, endowment, mutual fund — comes with a thick book of constraints: asset-class bounds, sector caps, tracking-error limits, liquidity floors, no-shorting rules. Risk officers will tell you these exist for prudence, and they do, but there is a mathematical reason underneath: the constraints make the feasible set compact, which makes the optimization *well-posed*. An unconstrained "maximize return" mandate is not just risky; it is mathematically ill-posed, with no attained optimum to target. The constraint book is, among other things, a compactness-manufacturing device. This is part of why a naive "100% in the highest-returning asset" is not the answer real allocators reach, as explored in [why not 100% equities](/blog/trading/quantitative-finance/jpm-why-not-100-equities).

**Numerical solvers and the "unbounded" status flag.** Commercial optimization solvers (the LP/QP engines under portfolio tools) return explicit status codes, and two of them are direct reports of the pathologies in this post: `UNBOUNDED` (the objective can be improved without limit — your feasible set was not bounded, no maximum exists) and `INFEASIBLE` (no point satisfies all constraints — the empty-set row of the matrix figure, nothing to optimize). A quant who reads these statuses through the lens of compactness diagnoses the problem instantly: `UNBOUNDED` means add a bound, `INFEASIBLE` means your constraints contradict each other and must be relaxed. The solver is literally reporting which leaf of the existence tree failed.

**Corner solutions and binding constraints.** When a linear objective is maximized over a compact polytope (the simplex, with linear constraints), the optimum always sits at a *vertex* — a corner where several constraints bind simultaneously. This is why mean-variance with a linear return objective and box constraints so often spits out concentrated, all-or-nothing weights: the attained maximum lives at a corner of the compact feasible set. Understanding that the optimum is *attained at the boundary* is exactly what motivates the shadow-price interpretation of the binding constraints, formalized by the [Lagrangian and KKT conditions](/blog/trading/math-for-quants/lagrangian-kkt-conditions-math-for-quants) — a constraint can only have a non-zero shadow price if the optimum actually reaches it, which requires the set to be closed.

## When this matters to you and further reading

If you ever run, build, or even just *trust* an optimizer, this is the layer of mathematics you are implicitly relying on every time it returns weights. The practical takeaway fits on an index card: before you believe an optimization's answer, audit two things. First, is the feasible set compact — closed (weak inequalities and equalities, boundary included) and bounded (every variable capped, leverage included)? Second, is the objective continuous (no fee cliffs, tax jumps, or rebate steps hiding in the score)? If both hold, the Extreme Value Theorem guarantees your problem has a real, attained answer and your solver is searching for something that genuinely exists. If either fails, you have diagnosed *in advance* why the output will be a supremum-mirage, a runaway, or a fragile cliff-edge — and the fix is almost always to add a constraint that restores compactness or to split the domain into continuous pieces.

This existence question is the silent prerequisite under the rest of the optimization curriculum. Once you know an optimum exists, the next questions are *how to find it* and *what its structure is*: that is [mean-variance optimization and the efficient frontier](/blog/trading/math-for-quants/mean-variance-efficient-frontier-math-for-quants) for the geometry of the solution, and the [Lagrangian and KKT conditions](/blog/trading/math-for-quants/lagrangian-kkt-conditions-math-for-quants) for how binding constraints shape and price that solution. The convergence logic of sequential compactness reappears whenever you reason about a sequence of decisions homing in on a long-run optimum, as in [the Kelly criterion and sequential betting](/blog/trading/quantitative-finance/kelly-criterion-sequential-betting-quant-interviews). And the reason real allocators never just dump everything into the single highest-returning asset — even though a naive unconstrained reading of "maximize return" suggests they should — is the same reason this post has been making all along: the well-posed, constrained, compact problem has a genuinely different and more sensible answer, as laid out in [why not 100% equities](/blog/trading/quantitative-finance/jpm-why-not-100-equities).

Compactness is not glamorous. It will never show up in a marketing deck. But it is the difference between an optimizer that hands you a portfolio and one that hands you a mirage, and once you can spot a non-compact feasible set or a discontinuous objective at a glance, you will catch a whole class of bugs before they ever reach production. This is educational material, not investment advice — but the habit of asking "does this problem even have an answer?" before trusting the answer is one of the cheapest and most valuable instincts a quant can build.
